# Building Magia トークン発行設計書

Monapartyのトークン発行パラメータには、**発行後に変更不可能な項目**があるため、各トークンの役割に応じて最適な設計を行う。設計を誤るとトークンの再発行が必要になるため、本設計書を正として扱う。

---

## 1. 不変パラメータの重要事項

調査の結果、特に以下の2つのパラメータは**初回の発行時に決定され、恒久的に変更不可能**であることが判明した。

| パラメータ | 日本語 | 変更可否 | Building Magiaでの重要性 |
|:---|:---|:---:|:---|
| `divisible` | 分割可能性 | **変更不可** | `false`にしないとNFTカードとして扱えない。`true`にしないと通貨として機能しない。 **最重要パラメータ。** |
| `callable` | 償還可能性 | **変更不可** | 今回は使用しない。 |

また、`locked`（発行数ロック）は一度ロックすると解除できない不可逆な操作である。

---

## 2. トークン別・発行パラメータ設計

上記を踏まえ、Building Magiaで発行する3種類のトークン（親トークン、NFTカード、ゲーム内通貨）のパラメータを以下のように設計する。

### 2.1. 親トークン（キャラクタークラス）

- **目的:** `GOHO` `KINNARA` `KARURA` という3つのクラス（種族）を定義する親。サブアセット（ビルカードNFT）を発行するための土台となる。
- **発行トークン:** `GOHO`, `KINNARA`, `KARURA`

| パラメータ | 値 | 理由 |
|:---|:---|:---|
| **種類** | Named Asset | 固有のクラス名を持つため。 |
| `quantity` | 1 | 親として1つ存在すれば十分。 |
| `divisible` | `false` | 通貨ではないため分割不要。 |
| `locked` | `true` | 発行数を1に固定し、意図しない追加発行を永久に防止する。信頼性の担保。 |
| `description` | `Parent token for the GOHO class.` | トークンの役割を明記。 |
| **発行費用** | 50 XMP / トークン | 合計150 XMP |

### 2.2. NFTビルディングカード（サブアセット）

- **目的:** ゲーム内でプレイヤーが所有し、バトルで使用する個別のNFTキャラクターカード。
- **発行トークン例:** `GOHO.SHIBUYASCRAMBLESQUARE`

| パラメータ | 値 | 理由 |
|:---|:---|:---|
| **種類** | Sub-asset | 親トークンに紐づく子トークンとして発行。 |
| `quantity` | 1 | 各ビルカードは唯一無二のNFTとするため。 |
| `divisible` | `false` | **NFTの必須条件。** Monacardとして画像と紐付けるためにも分割不可にする必要がある。 |
| `locked` | `true` | 発行数を1に固定し、NFTとしての希少性と唯一性を保証する。 |
| `description` | `json` | 9方位属性、ビル写真のIPFSハッシュ、その他メタデータをJSON形式で格納。 |
| **発行費用** | 25 XMP / ビル | |

### 2.3. ゲーム内通貨（イベント報酬）

- **目的:** イベント報酬やゲーム内エコノミーで使用する代替可能な通貨（Fungible Token）。
- **発行トークン:** `KEKKAI`

| パラメータ | 値 | 理由 |
|:---|:---|:---|
| **種類** | Named Asset | ゲーム内通貨として固有の名前を持つため。 |
| `quantity` | 100,000,000 | 初期発行量。多めに設定。 |
| `divisible` | `true` | **通貨の必須条件。** 0.00000001単位での送金や支払いを可能にする。 |
| `locked` | `false` | **将来の追加発行を可能にするため、ロックしない。** イベント報酬などで供給が枯渇した場合に追加発行できるよう余地を残す。 |
| `description` | `In-game currency for Building Magia.` | トークンの役割を明記。 |
| **発行費用** | 50 XMP | |

---

## 3. 発行手順

1. **親トークン発行:** Monapalette等のツールを使い、上記2.1の設計で`GOHO`, `KINNARA`, `KARURA`の3つを発行する。
2. **ゲーム内通貨発行:** 同様に、2.3の設計で`KEKKAI`を発行する。
3. **NFTビルカード発行:** 
    1. ビル写真をIPFSにアップロードし、IPFSハッシュを取得。
    2. 9方位属性をランダム生成。
    3. 上記を含むJSONメタデータを作成。
    4. Monapaletteを使い、2.2の設計でサブアセット（例: `GOHO.SHIBUYASCRAMBLESQUARE`）を発行。DescriptionにJSONメタデータを書き込む。
